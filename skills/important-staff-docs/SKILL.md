# Important Staff Docs

## Quick Workflow

1. Classify the product: order, report, estimate, brief, matrix, annex, or correspondence.
2. Identify owner and reviewers: S-3/XO for orders, section head for estimates, commander/XO for decisions, special staff for their annexes.
3. Keep the product training-safe unless the user is in an approved environment.
4. Use the smallest useful product: WARNO to start planning, OPORD/CONOP to direct, FRAGO to change, SITREP to update, AAR to learn.
5. Pull detailed templates from `references/staff-products.md` only when a concrete scaffold is needed.

## Product Routing

- WARNO: alert the staff/subordinates and start parallel planning.
- OPORD/CONOP: direct execution or explain the concept with five-paragraph discipline.
- FRAGO: state only what changed from the base order.
- SITREP: report current status, changes, risks, and next 24-72 hours.
- Running estimate: section-owned current situation, assumptions, risks, supportability, and decisions.
- CUB: brief command on posture, friction, decisions, and near-term support requirements.
- CPB: Civil Preparation of the Battlespace. Use only for Civil Affairs/G-9 problems; otherwise prefer CUB, decision brief, running estimate, FRAGO, or CONOP.
- AAR: capture standards, observations, sustains, improves, and corrective actions.
- Annex/appendix: put section-specific detail where it supports the base order without bloating it.

## Reference Discipline

Use official/public references first:

- `MCWP 5-10 Marine Corps Planning Process`
- `MCDP 5 Planning`
- `MCTP 3-30A Command and Staff Actions`
- `CJCSM 3150.05F Joint Reporting System Situation Monitoring Manual`
- `MCO 1553.3C Unit Training Management`
- `SECNAV M-5216.5` and `MCO 5216.20B` for correspondence

Do not invent annex lettering, approval authority, reporting formats, or current publication status. If the exact format matters, cite the source and tell the user to verify the current local SOP or governing order.
