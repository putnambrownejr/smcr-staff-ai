# Dashboard Integrity and Navigation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the approved Watch, demo isolation, template integrity, FitReps navigation, agent-ID, CRT EGA icon, and Initial Counseling improvements as independently tested changes.

**Architecture:** Keep backend truth in existing Pydantic schemas and local stores, then expose it through the current FastAPI routes. Preserve the compiled dashboard workflow by expressing UI changes in `scripts/patch_dashboard_bundle.py` and regenerating `app/static/dashboard/index.html`. Reuse User Docs Generation records for Initial Counseling and store only explicit FitRep IDs for links.

**Tech Stack:** Python 3.12, FastAPI, Pydantic, pytest, vanilla JavaScript embedded in the dashboard bundle, PowerShell, stdlib PNG/ICO generator.

## Global Constraints

- Runtime remains local-first and UNCLASSIFIED-only.
- Missing publication dates are omitted; no dates are invented.
- Demo projects are hidden only through explicit `.smcr-project.json` metadata and are never deleted by the toggle.
- Missing or invalid project metadata defaults to non-demo so user work is not hidden.
- Every system-template catalog entry resolves to one checked-in Markdown file.
- Initial Counseling may be unlinked; it never auto-creates a FitRep or silently changes FitRep evaluation fields.
- Dashboard bundle changes are made through `scripts/patch_dashboard_bundle.py`, not by hand-editing generated markup.
- Icon assets are regenerated by `scripts/generate_app_icon.py`, not edited by hand.
